## Why

학지사 도서를 빠르게 찾아보고 비슷한 책을 둘러볼 가벼운 진입점이 없다. 정식 커머스/검색 인프라를 구축하기 전에, 카탈로그를 키워드로 검색하고 관련 도서를 추천하는 최소 기능을 빠르게 검증한다.

## What Changes

- 학지사 도서 카탈로그를 담는 단일 테이블(`contents`)과 시드 데이터(50~100권)를 도입한다.
- 제목·저자·소개 텍스트에 대한 **키워드 부분일치 검색**(`ILIKE`)을 제공한다.
- **분야(category)별 둘러보기**와 **도서 상세 페이지**를 제공한다.
- 상세 페이지에서 **같은 분야의 다른 책**을 추천한다(머신러닝/벡터 없음).
- 화면은 3개(홈/검색, 검색 결과, 도서 상세)로 한정한다.

명시적 비목표(Non-goals): 로그인·회원, 행동 로그·협업필터링, 벡터/의미 검색, 결제·장바구니, 관리자 CMS, 실시간 크롤링.

## Capabilities

### New Capabilities
- `content-search`: 학지사 도서 카탈로그의 키워드 검색, 분야별 둘러보기, 도서 상세 조회, 같은 분야 기반 관련 도서 추천을 포함한다.

### Modified Capabilities
- (없음 — 신규 프로젝트)

## Impact

- **스택**: Next.js(App Router) + Supabase(Postgres).
- **신규 코드**: `contents` 테이블 스키마/시드, 검색·추천 쿼리, 3개 페이지(홈/검색결과/상세) 및 검색 API 경로.
- **데이터**: 시드 JSON/CSV 50~100권을 수동 적재(향후 정식 카탈로그 연동 시 교체).
- **확장 여지**: 검색 정밀도 부족 시 `pg_bigm` 또는 pgvector로 교체 가능(스키마 호환).
