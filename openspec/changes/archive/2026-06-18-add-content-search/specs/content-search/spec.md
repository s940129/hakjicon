## ADDED Requirements

### Requirement: 도서 카탈로그 데이터 모델
시스템은 학지사 도서를 단일 `contents` 테이블에 저장(MUST)해야 하며, 각 도서는 `id`, `title`, `author`, `category`, `description`, `cover_url`, `isbn`, `link` 필드를 가진다. `title`, `author`, `category`는 비어 있을 수 없다.

#### Scenario: 시드 도서 적재
- **WHEN** 시드 데이터(50~100권)를 Supabase `contents` 테이블에 적재한다
- **THEN** 각 행은 `title`, `author`, `category`가 채워진 상태로 저장된다
- **AND** 표지가 없는 도서는 `cover_url`이 비어 있어도 적재에 실패하지 않는다

### Requirement: 키워드 검색
시스템은 사용자가 입력한 키워드로 `title`, `author`, `description`에 대해 대소문자 무시 부분일치(`ILIKE '%키워드%'`) 검색을 수행하고, 일치하는 도서 목록을 반환(MUST)해야 한다.

#### Scenario: 일치하는 도서가 있는 검색
- **WHEN** 사용자가 검색어 "상담"을 입력한다
- **THEN** 제목·저자·소개 중 어느 하나라도 "상담"을 포함하는 도서들이 결과로 반환된다
- **AND** 각 결과 항목은 제목, 저자, 표지를 표시한다

#### Scenario: 일치하는 도서가 없는 검색
- **WHEN** 사용자가 어떤 도서와도 일치하지 않는 검색어를 입력한다
- **THEN** 빈 결과와 "검색 결과 없음" 안내가 표시된다

#### Scenario: 빈 검색어
- **WHEN** 사용자가 검색어 없이 검색을 시도한다
- **THEN** 검색을 실행하지 않고 홈의 기본 상태(분야 둘러보기)를 유지한다

### Requirement: 분야별 둘러보기
시스템은 홈에서 도서 분야(category) 목록을 제공하고, 분야를 선택하면 해당 분야의 도서 목록을 보여줘야(MUST) 한다.

#### Scenario: 분야 선택
- **WHEN** 사용자가 "상담" 분야를 선택한다
- **THEN** `category = '상담'`인 도서 목록이 표시된다

### Requirement: 도서 상세 조회
시스템은 도서 식별자로 단일 도서의 상세 정보를 조회해 표지, 제목, 저자, 분야, 소개, 학지사 원본 링크를 표시(MUST)해야 한다.

#### Scenario: 존재하는 도서 상세
- **WHEN** 사용자가 검색 결과에서 도서를 선택한다
- **THEN** 해당 도서의 표지·제목·저자·분야·소개와 학지사 원본 페이지로 이동하는 링크가 표시된다

#### Scenario: 존재하지 않는 도서
- **WHEN** 사용자가 존재하지 않는 도서 식별자로 상세를 요청한다
- **THEN** "도서를 찾을 수 없음" 안내가 표시된다

### Requirement: 같은 분야 관련 도서 추천
시스템은 도서 상세 페이지에서 동일한 `category`를 가진 다른 도서를 최대 5권 추천(MUST)해야 하며, 현재 보고 있는 도서는 제외한다.

#### Scenario: 같은 분야 추천 노출
- **WHEN** 사용자가 "상담" 분야 도서의 상세 페이지를 본다
- **THEN** `category = '상담'`이고 현재 도서가 아닌 도서 중 최대 5권이 "같은 분야 추천"으로 표시된다

#### Scenario: 같은 분야 도서가 부족한 경우
- **WHEN** 같은 분야에 현재 도서 외 다른 도서가 없다
- **THEN** 추천 영역을 비워 두거나 표시하지 않는다(오류를 발생시키지 않는다)
